## Summary

beta-7 connects the post-simulation pipeline to real simulation exits and exports
analysis, report, and dashboard artifacts to disk through a deterministic, auditable
export path.  It does not invent new subsystems.  It makes the existing ones reachable,
testable, and honest.

## Hash Capture Method

Golden hashes were computed using `--capture` mode on the canonical beta-7 build:

```
.\build_test\Debug\qa_golden_tests.exe --capture
```

Each `CAPTURE:<name>:<hash>` line is the output of `StructureCanonicalizer::compute_hash()`
applied to the FIRE-relaxed structure.  Hashes are deterministic on the same build and
platform.  To re-verify after a clean rebuild, run `--strict` and confirm 17/17 pass.

Deferred crystals use `SKIP_GOLDEN_HASH("reason", "target_version", blocks_release)`
which is documented in `apps/qa_golden_tests.cpp` and treated as an explicit skip
(not a failure) by `validate_strict()`.
