## 1. Abstract

VSEPR-SIM version 5 is the penultimate version iteration, finally touching on true macro-scale properties. Beta-6 fills in several mechanisms for lower-level verification before doing so — implications initially skipped for simplification in previous versions but impossible to ignore when moving onto very complex systems in coming developments (v5-beta.8).

Beta-6 is the **instrumentation release**. It does not claim to be macro. It claims to have built the instruments required before macro can be trusted.

The result is a working trajectory-to-inference pipeline that can measure structural change, surface interaction, diffusion behavior, and bead/powder packing from trajectory data — without storing derived material properties in simulation state. This distinction is the entire project philosophy in practice.

## 3. Core Design Rule — Preserved Across All New Systems

The strongest result from this development block:

> **No inferred macro property is written back into State.**

The system respects four clean layers:

| Layer | Role |
|---|---|
| **State** | Position, velocity, identity, dynamic truth |
| **xyzFull** | Trajectory evidence |
| **Analysis** | Measures behavior from trajectory |
| **Inference** | Estimates macro behavior from measured behavior |

Quantities that are explicitly **not** stored in State:

- diffusion coefficient
- effective diffusivity by axis
- transport class
- porosity
- bulk density
- packing class
- compressibility proxy
- permeability proxy
- sintering readiness
- formation energy proxy

These are analysis-only outputs. The code compiles this guarantee — there is no field in `atomistic::State` for any of them. That is not accidental. It is the architecture.

## 4. Measurement Backbone — Completed Components

The following measurement infrastructure was validated during beta-6:

| Component | Role |
|---|---|
| `vsepr::Vec3` authoritative state vector | Single canonical position/velocity type |
| Eigen bridge (isolated at analysis boundaries) | Matrix ops without contaminating core state |
| Kabsch / RMSD analysis | Rotation-invariant structural residual |
| Stationarity / settling gates | Convergence check before inference |
| Translation / rotation / deformation distinction | Separates motion types from structural change |

**Key validation result:**

- Kabsch RMSD ignores rigid translation and rotation
- Kabsch RMSD responds to true non-rigid deformation
- Mean displacement captures raw motion
- Structural residual captures reference-site mismatch

This gives the project a defensible way to separate *moved*, *rotated*, *deformed*, *relaxed*, and *destabilized* — a distinction required before any macro inference can be credible.

## 5. Task 9 — Crystal Imperfection Emergence

**Status: PASS**

Crystal imperfection detection validated across SC, FCC, and BCC lattices with the following variants:

| Variant | Detected behavior |
|---|---|
| ideal | Low RMSD, no occupancy mismatch, no excess atoms |
| interstitial | N_excess = +1 |
| vacancy | N_excess = −1, occupancy mismatch |
| substitutional | structure_class = ideal, identity_class = substitutional_site |
| thermal_jitter | Displacement disorder, no false count defect |
| localized_impulse | Local relaxation / disruption |
| multi_site | Broad disorder / instability |

**Primary achievement:** Substitutional defects are separated into structural and identity channels. The analyzer correctly reports:

```
structure_class = ideal_crystal
identity_class  = substitutional_site
```

A substitution can be geometrically clean while identity-defective. The code does not panic and call everything unstable.

**Recommended future refinement (beta-7):** Expand classifier vocabulary to include `thermal_disorder`, `localized_relaxation`, `broad_disorder`, `atom_excess`, `atom_deficit`, `substitutional_site`. The numerical detectors are already stronger than the current labels.

**Status: PASS**

Surface interaction behavior validated from trajectory only across six cases:

|---|---|

| medium normal velocity | no_interaction |
| grazing | no_interaction |
| angled impact | no_interaction |
| high normal velocity | temporary_residence / contact |

**Key result:** The stream case detected embedded behavior without storing `surface_damage` or `embedded` as state truth. Classification is derived entirely from trajectory displacement and contact duration.

**Recommended future refinement (beta-7):** Classifier priority should enforce event hierarchy:
```
embedded_event > ejection_event > temporary_residence > contact_no_capture > no_interaction
```

## 11. Known Limitations and Honest Notes

### Energy drift in NVE Lennard-Jones tests
All 12 diffusion cases flag `invalid_energy_drift` and `valid_for_macro_inference = no`. This is correct. NVE integration without PBC or thermostat on a small SC crystal drifts. The flag works. D_eff values are still computed and reported — they reflect kinetic behavior, not thermodynamic equilibrium. The `valid_for_macro_inference` gate prevents them from being treated as converged transport coefficients.

For physically valid D_eff values, the system requires a thermostat (Langevin already exists in `actions_form.cpp`) and PBC (supported in `atomistic::Box`). That wiring is a beta-7 task.

### BCC Argon at A=3.30 Å
The BCC case has ΔU_proxy = +11.9 kcal/mol vs SC reference. This is because A=3.30 Å is well inside the LJ repulsive wall for Ar (σ=3.4 Å). The BCC test correctly shows overcompressed-repulsive behavior. It is not a physical BCC Ar ground state. It is a structural comparison case. The label `ideal_reference` for BCC refers to the geometric reference classification, not a claim of thermodynamic stability.

### Interstitial ΔU_proxy = +34.6 kcal/mol
The interstitial atom is placed at an off-lattice position in the repulsive core of its neighbors. The large ΔU correctly reflects that a naive geometric interstitial injection is high-energy. A FIRE-relaxed interstitial would show a smaller (and physically meaningful) formation energy. FIRE relaxation is available in the existing integrator stack and is a beta-7 enhancement target.

### Activation trend returns `insufficient_data`
The 4-point thermal sweep produces a 130× D range but all cases are `invalid_energy_drift`. The Arrhenius fit gate correctly excludes them. To get a real activation trend, the cases must be thermostatted. The infrastructure is correct. The physics input is not yet calibrated.
